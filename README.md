#  Catch-A-Bite

##  프로젝트 소개
**Catch-A-Bite**는 기존 배달 플랫폼의 구조적 문제를 해결하기 위해 기획된 대안 배달 플랫폼입니다.

###  개발 배경
기존 배달 서비스(배달의민족, 요기요 등)는  
- 높은 중개 수수료  
- 광고비 강제  
- 배달료 전가  
로 인해 점주, 소비자, 라이더 모두에게 부담을 주고 있습니다.

###  목표
- 점주 수익성 개선
- 소비자 가격 부담 완화
- 배달 생태계 구조 개선

---

##  프로젝트 기획

###  문제 정의
- 점주 마진 25~35% 감소
- 플랫폼 중심 구조로 참여자 모두에게 불리

###  기획 방향
- 투명한 수수료 구조
- 데이터 통합 기반 UX 개선
- 사용자 / 사업자 / 라이더 3자 플랫폼 설계

###  핵심 전략
- DTO 기반 데이터 통합 → 프론트 로직 단순화
- 메뉴/옵션 구조 확장 가능 설계

##  트러블 슈팅

###  문제 1: 결제 API 연동 오류
- PORTONE 키 생성 시 Order ID 필요
- Lazy Loading으로 데이터 조회 실패

###  해결
- Fetch Type을 **EAGER**로 변경

---

###  문제 2: 배달 상태 동기화 문제
- DeliveryStatus와 OrderStatus가 서로 달라
  배달 상태가 정상적으로 연동되지 않음

###  해결
- enum 값 통일
- front / back 상태 값 일치화
- 명령 기반 방식 적용
- 하나의 메서드에서 상태 전이 처리
- DeliveryStatus + OrderStatus 동시 진행 구조로 개선
