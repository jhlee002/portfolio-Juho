문제 상황

배달 진행 과정에서 DeliveryStatus(라이더)와 OrderStatus(주문고객/점주)가 호환되지 않아 배달 단계가 연동되지 않는 문제가 발생했습니다.

원인

배달 상태를 enum 값으로 관리하고 있었지만, front와 back에서 사용하는 상태 값이 서로 다르게 설정되어 있었습니다.

해결 방법
  • DeliveryStatus와 OrderStatus에서 사용할 enum 값을 통일
  • front와 back의 상태 값을 동일하게 수정
  • back-end에서 명령 기반 방식을 적용
  • 하나의 메서드로 두 가지 상태 전이가 함께 이루어지도록 수정
  • DeliveryStatus와 OrderStatus가 병행 진행되도록 개선
협업 및 통합 테스트 경험을 통한 개선점

Catch-A-Bite 프로젝트에서는 Rider 기능의 Back-End 개발을 주로 담당하였습니다. 기능 구현 과정에서 API 설계와 비즈니스 로직 개발에 집중하였지만, Front-End와의 실제 연동 테스트 및 화면 단위 검증을 충분히 병행하지 못한 아쉬움이 있었습니다.

프로젝트 후반부에서 일부 기능의 데이터 구조 및 응답 형식 차이로 인해 연동 과정에서 추가 조정이 필요했고, 이를 통해 다음과 같은 점의 중요성을 크게 느꼈습니다.

개선점
  • Front-End와 Back-End 간의 초기 API 명세 공유
  • 기능 개발 중간 단계의 지속적인 연동 테스트
  • 상태값(Enum), DTO 구조, 응답 포맷의 사전 합의
  • 구현 완료 이후가 아닌 개발 단계부터의 협업 검증

이 경험 이후에는 Back-End 개발을 진행하더라도 단순 API 구현에 그치지 않고, 실제 화면 흐름과 사용자 동작까지 고려하며 Front-End 개발자와 지속적으로 소통하는 개발 방식을 중요하게 생각하게 되었습니다.
